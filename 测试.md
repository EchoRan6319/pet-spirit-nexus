## 完整用户旅程链路

```
① 用户打开小程序
      ↓
② 首页填写宠物档案（名字/物种/特征）
      ↓ POST /api/user/create
③ 后端创建 user + pet 记录到 SQLite
      ↓ 返回 userId
④ 前端存储 userId 到 Storage → 跳转聊天页
      ↓
⑤ 用户发送消息 "我今天好想你"
      ↓ POST /api/chat { userId, 
      message }
⑥ 后端查询档案 → 构建 userContext → 调用 
LLM
      ↓
⑦ LLM 返回:
   ├─ type:"text" → 显示宠物回复气泡
   ├─ type:"action" + hug/heartbeat → 
   触发震动+光晕动效
   ├─ type:"memory_update" → 后台静默更
   新 DB 记忆标签
   └─ type:"action" + lock_app (第49
   天) → 显示祝福语 → 3秒后跳转纪念碑
      ↓
⑧ 纪念碑页面：
   - 展示完整档案摘要
   - 显示最后告别语
   - 统计陪伴天数和回忆数量
   - 锁定只读状态（永久珍藏）
```

1. LLM API Key 配置 : 必须在 backend/.env 中填入有效的 API Key（推荐 DeepSeek 或智谱）
2. 跨域问题 : H5 模式下前端端口 5173 → 后端 3000，需确保后端 CORS 中间件已启用（已在 index.js 配置）
3. 真机测试 : 微信震动 API ( uni.vibrateShort() ) 需要在真机或微信开发者工具中测试，H5 浏览器不支持
