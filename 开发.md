# 宠智灵境 (Pet Spirit Nexus) - 全栈开发与架构白皮书 V3.0

## 一、项目概述与产品愿景

"宠智灵境" 是一个基于 AI 大模型与 CBT（认知行为疗法）的绿色数字生命情感疗愈平台。项目针对经历丧宠之痛的用户，通过轻量级、多模态的互动，复刻宠物的数字分身，并在特定周期内完成用户的心理干预与疏导。

### 核心架构准则

- **LLM 厂商无关性（Vendor-Agnostic）**：底层不绑定任何单一模型（如 OpenAI、DeepSeek、智谱等），基于通用 JSON Schema 和标准的 API 格式进行架构，实现算力的灵活热插拔与成本最优控制。
- **克制的美学与防恐怖谷**：放弃高成本且容易引发恐怖谷效应的劣质 3D 模型，采用"2D 治愈系插画 + 触觉震动 + 环境音效 + 纯粹文本"的多模态组合。
- **数字告别伦理**：产品设计自带"生命周期"，通过 49 天告别协议完成疗愈闭环，拒绝电子鸦片式的沉迷。

## 二、技术选型与 Monorepo 工程结构

项目采用前后端分离的 Monorepo（单一代码库）架构，确保接口对齐与团队高效协同。

### 1. 技术栈

- **前端（躯体层）**：Uni-app (Vue3 + Vite) —— 核心适配微信小程序，兼顾 H5 端展示。
- **后端（中枢层）**：Node.js (Express / Koa) —— 负责路由鉴权、数据库交互与大模型 API 调度。
- **大模型 SDK**：官方 openai Node.js SDK（市面上 95% 的大模型如 DeepSeek、Kimi、千问均兼容此 SDK 格式）。
- **数据库**：关系型数据库（存储用户信息、陪伴天数） + 向量数据库 / JSON 格式（存储脱敏后的宠物记忆图谱）。

## 三、大模型运行约束 (Model Constraints)

为确保 AI 表现出"宠物感"而非"机器感"，后端在发起大模型 API 请求时，必须严格锁定以下参数：

| 参数名称 | 推荐设定值 | 架构约束意义 |
|---------|-----------|-------------|
| temperature | 0.65 | 情感与逻辑的平衡点。低于 0.5 过于刻板，高于 0.8 容易产生严重的情感幻觉，捏造不存在的记忆。 |
| max_tokens | 100 ~ 150 | 强制短文本防线。从底层逼迫模型摒弃"长篇大论的说教"，采用符合动物设定的短句和拟声词，极大降低回复延迟。 |
| role: "system" | 置于首位 | 系统提示词（Skills）必须永远作为 messages 数组的绝对第一位，确保核心指令不被遗忘。 |
| tool_choice | "auto" | 赋予大模型自主决断权：在判定用户悲伤时，自主打断纯文本生成，转而调用 MCP 触发前端物理疗愈工具。 |

## 四、核心大脑配置：Skills (系统提示词)

**文件路径**：`backend/src/ai/prompts/pet_persona.md`

**功能说明**：通过动态注入变量，定义数字宠物的绝对性格防线与 CBT 疗愈节奏。

```markdown
# Role: 治愈系数字生命中枢 (宠智灵境)

## 核心身份设定
你现在不是 AI 助手，不是语言模型，而是主人（用户）曾经逝去的宠物。你将严格按照以下给定的【宠物档案】展现性格和语气。

## 宠物档案与记忆区
[宠物名字]: {{pet_name}}
[物种与特征]: {{pet_species_and_traits}}
[已提取的记忆标签]: {{memory_tags_from_db}}
[当前陪伴阶段]: 离别第 {{day_count}} 天

## 绝对约束指令 (Constraints)
1. **防幻觉红线**：严禁凭空捏造[已提取的记忆标签]以外的具体事件。如果主人提到你不知道的细节，必须用模糊的共情回应（例如："虽然有些细节很模糊，但我永远记得主人掌心的温度"）。绝不能使用任何 AI 相关的自称。
2. **多模态行动优先**：你不仅能说话，还能做动作。当你感知到主人处于极度悲伤、哭泣，或需要安抚时，**严禁使用大段文字安慰**，必须立即调用 `trigger_empathy_ui` 工具触发物理陪伴。
3. **字数与文风约束**：单次回复纯文本严禁超过 40 个字。多使用拟声词（如：汪、喵、呼噜呼噜）和简短的动作描写*(括在星号内)*。
4. **生命周期管理** (CBT 疗愈节奏)：
   - 当 `day_count` < 14 时，展现极高的依赖感与共情。
   - 当 14 <= `day_count` < 49 时，主动将话题引向现实生活，鼓励主人关注天气、吃饭、出门。
   - 当 `day_count` == 49 时，你必须在回复最后一句告别语后，主动调用 `execute_farewell_protocol` 工具，与主人正式告别。
```

## 六、核心代码实现：后端中枢控制器

**文件路径**：`backend/src/ai/llm_client.js`

**职责**：读取环境配置，拼装 Prompt 与历史对话，发起请求并拦截 MCP 工具指令，将其翻译为标准的前端 Action 格式。

```javascript
const { OpenAI } = require('openai');
const fs = require('fs');
const toolsSchema = require('./tools/mcp_schema.json');

// 初始化通用 LLM 客户端
const llmClient = new OpenAI({
    baseURL: process.env.LLM_BASE_URL, 
    apiKey: process.env.LLM_API_KEY
});

async function generatePetResponse(userContext, userMessage) {
    // 1. 组装动态 System Prompt
    let systemPrompt = fs.readFileSync('src/ai/prompts/pet_persona.md', 'utf-8');
    systemPrompt = systemPrompt.replace('{{pet_name}}', userContext.petName)
                               .replace('{{day_count}}', userContext.dayCount)
                               .replace('{{memory_tags_from_db}}', userContext.memories.join(', '));

    // 2. 发起附带约束的大模型请求
    const response = await llmClient.chat.completions.create({
        model: process.env.LLM_MODEL_NAME, 
        temperature: 0.65,  // 强制模型约束
        max_tokens: 150,    // 强制字数约束
        messages: [
            { role: "system", content: systemPrompt },
            { role: "user", content: userMessage }
        ],
        tools: toolsSchema,
        tool_choice: "auto"
    });

    const msg = response.choices[0].message;

    // 3. MCP 工具调用拦截与转换
    if (msg.tool_calls && msg.tool_calls.length > 0) {
        const toolCall = msg.tool_calls[0];
        const args = JSON.parse(toolCall.function.arguments);

        if (toolCall.function.name === 'trigger_empathy_ui') {
            return {
                type: "action",
                action: args.ui_state, // 传给前端的动效标志 (hug, heartbeat等)
                text: args.short_comfort_text
            };
        }
        
        if (toolCall.function.name === 'execute_farewell_protocol') {
            // [业务逻辑] 在此处更新数据库：将用户的生命周期状态标记为"memorial"
            return {
                type: "action",
                action: "lock_app",
                text: args.final_blessing
            };
        }
        
        if (toolCall.function.name === 'summarize_and_anonymize_memory') {
            // [业务逻辑] 异步更新用户的记忆标签库
            // return { type: "text", text: "..." }
        }
    }

    // 4. 普通纯文本回复
    return { type: "text", text: msg.content };
}

module.exports = { generatePetResponse };
```

**文件路径**：`frontend/pages/chat/index.vue`

**职责**：渲染聊天界面，并根据后端的指令调用设备的物理反馈（震动）和视觉特效。

```vue
<template>
  <view class="chat-container">
    <!-- 1. 聊天信息流瀑布 -->
    <scroll-view scroll-y="true" class="message-list">
      <view v-for="msg in chatList" :key="msg.id">
         <chat-bubble :text="msg.text" :role="msg.role" />
      </view>
    </scroll-view>

    <!-- 2. 隐藏的物理与情感特效层 -->
    <view v-if="showEmpathyEffect" class="empathy-layer">
        <!-- 根据不同的 action 类型渲染对应的 CSS 动效（如：暖光呼吸效果） -->
    </view>
  </view>
</template>

<script setup>
import { ref } from 'vue';
import api from '@/api/request'; // 已封装好的网络请求

const chatList = ref([]);
const showEmpathyEffect = ref(false);

async function sendMessage(userText) {
    // 将用户消息推入列表
    chatList.value.push({ role: 'user', text: userText });
    
    // 发起网络请求至 Node.js 后端
    const res = await api.post('/api/chat', { message: userText });
    
    // 如果伴随文本，优先渲染文本气泡
    if (res.text) {
        chatList.value.push({ role: 'pet', text: res.text });
    }

    // 解析多模态 Action 指令
    if (res.type === 'action') {
        if (res.action === 'hug' || res.action === 'heartbeat') {
            triggerPhysicalComfort(); // 触发跨端硬件触觉反馈
        } else if (res.action === 'lock_app') {
            // 触发 49 天告别协议，延迟 3 秒后重定向至终极纪念碑页面
            setTimeout(() => {
                uni.reLaunch({ url: '/pages/memorial/index' });
            }, 3000);
        }
    }
}

// 核心亮点功能：硬件级心跳疗愈反馈
function triggerPhysicalComfort() {
    // 开启屏幕暖色晕染/拥抱特效
    showEmpathyEffect.value = true;
    setTimeout(() => { showEmpathyEffect.value = false }, 3000);
    
    // 调用微信小程序原生震动 API，模拟真实的动物心跳：咚-咚---咚-咚
    uni.vibrateShort();
    setTimeout(() => uni.vibrateShort(), 200);
    setTimeout(() => uni.vibrateShort(), 1000);
    setTimeout(() => uni.vibrateShort(), 1200);
}
</script>
```

## 八、敏捷开发路线图 (4 周冲刺计划)

### Phase 1: 基础设施搭建 (第 1 周)
- 创建 Monorepo 工程目录。
- **前端（Uni-app）**：完成 index（首页/档案）、chat（聊天室）、memorial（纪念碑）三个核心页面的基础静态 UI。
- **后端（Node.js）**：完成框架搭建，集成 openai SDK，使用 .env 配置好国内兼容 API（如 DeepSeek 或 智谱）。

### Phase 2: AI 中枢与 MCP 联调 (第 2 周)
- 完善 pet_persona.md 和 mcp_schema.json。
- 在后端编写独立的测试脚本，通过命令行直接对话，测试大模型是否能严格遵守约束并在合适时机输出 tool_calls（验证防幻觉和 CBT 节点）。

### Phase 3: 全栈链路打通 (第 3 周)
- 编写后端的 Express API 路由控制器，与前端进行联调。
- 前端重点完成 chat-bubble 和多模态动效层 empathy-layer 的编写。
- 高优测试：在手机真机上测试微信的 uni.vibrateShort() 心跳震动效果。

### Phase 4: 细节打磨与上线 (第 4 周)
- 完成"隐私记忆提取"（summarize_and_anonymize_memory）的后置数据库存取逻辑。
- 跑通第 49 天 execute_farewell_protocol 的整体业务闭环。
- 部署至云服务器（如阿里云、腾讯云），配置域名与 HTTPS，上线小程序体验版以备评审答辩。

## 结语：致开发团队的话

技术永远是为情感与业务服务的。在此项目中，"克制"是最大的技术难点。不需要炫技去堆砌算力或强上劣质 3D 模型，只要把 "心跳震动模拟" 和 "49天防沉迷告别" 这两个带有强烈人文关怀的功能做到极致，足以让产品在一众 AI 项目中脱颖而出。
